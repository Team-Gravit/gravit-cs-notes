## 응용 계층(Application Layer)

**OSI 7계층**의 최상위 계층으로, 사용자가 직접 네트워크 서비스를 이용할 수 있게 하는 계층이다. 애플리케이션과 하위 계층 간의 인터페이스 역할을 하며, 네트워크에서 데이터를 주고받기 위한 규칙을 정의한다.

<br>

### 1. 주요 기능

- 네트워크를 통한 서비스 제공 및 접근 제어
- 클라이언트와 서버 간의 **요청**(Request)과 **응답**(Response) 처리
- 하위 계층에서 받은 데이터를 사용자에게 표시 가능한 형태로 변환
- 각 프로토콜마다 고유한 포트 번호를 사용함(관리자가 변경 가능)

<br>

### 2. 주요 프로토콜

| 프로토콜 | 설명 |
| --- | --- |
| **HTTP** | 웹페이지 전송용 프로토콜, 클라이언트-서버 간 요청/응답 방식 |
| **DNS** | 도메인 이름을 IP 주소로 변환하는 이름 해석 시스템 |
| **SMTP** | 이메일 송신용 프로토콜 |
| **FTP** | 파일 전송 프로토콜 |
| **SSH** | 원격 접속 시 데이터를 암호화하여 안전하게 통신하는 프로토콜 |
| **Telnet** | 원격 접속용 프로토콜이지만 암호화되지 않아 보안에 취약함 |

<br>

### 3. HTTP(Hypertext Transfer Protocol)

웹 통신을 담당하는 프로토콜로, 클라이언트(브라우저)와 서버 간의 **요청**과 **응답**을 처리한다.

<br>

**동작 과정**

- 사용자가 **URL**을 입력하면 브라우저가 서버에 **HTTP 요청**을 보냄
- 서버는 **HTML**, 이미지 등 필요한 데이터를 응답함

<br>

**특징**

- **비연결성(Connectionless)**: 요청마다 독립적으로 처리되며 요청 후 연결이 종료됨
- **상태비저장성(Stateless)**: 이전 요청 정보를 저장하지 않음
- 대표 메서드: **GET**, **POST**, **PUT**, **DELETE** 등

<br>

### 4. DNS(Domain Name Service)

도메인 이름을 **IP 주소**로 변환하는 프로토콜이다. 사람이 읽기 쉬운 주소를 컴퓨터가 이해할 수 있는 숫자 주소로 변환한다.

<br>

**동작 과정**

1. 사용자가 웹사이트 주소를 입력함
2. **DNS 서버**가 해당 도메인의 IP를 찾아 응답함
3. **HTTP**가 이 IP로 통신을 수행함

<br>

### 5. SMTP(Simple Mail Transfer Protocol)

이메일을 전송하는 프로토콜로, 메일 클라이언트에서 메일 서버로 메일을 보낼 때 사용한다. 메일 서버 간에도 **SMTP**로 메시지가 전달된다.

**특징**

- 응용 계층에서 동작함
- 포트 **25**(보안 연결 시 **587**, **465**) 사용

<br>

### 6. FTP(File Transfer Protocol)

파일을 업로드·다운로드할 때 사용하는 프로토콜이다.

**특징**

- 클라이언트가 서버에 접속해 파일을 전송하거나 가져올 수 있음
- **명령 채널**(제어)과 **데이터 채널**(전송)을 분리해 통신함
- 보안 강화를 위해 **FTPS**나 **SFTP**(SSH 기반)도 사용함

<br>

### 7. SSH(Secure Shell)

원격 시스템에 안전하게 접속하기 위한 **암호화 통신 프로토콜**이다.

**특징**

- **Telnet**과 달리 모든 데이터가 암호화되어 전송되므로 보안성이 높음
- 서버 관리, 원격 명령 실행, 파일 전송(**SCP/SFTP**)에 사용함
- 기본 포트는 **22번**
- **공개키 인증**을 통해 안전하게 접속할 수 있음
