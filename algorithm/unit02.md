## 공간 복잡도

**공간 복잡도**는 알고리즘이 문제를 해결하기 위해 **얼마나 많은 메모리를 사용하는가**를 분석하는 척도이다.

입력 크기 **n**이 커질수록 추가로 필요한 메모리 양이 얼마나 증가하는지를 평가한다.

<br>

### 1. 공간 복잡도 구분

- **고정 공간**: 입력 크키와 관계없이 항상 동일한 크기의 메모리를 사용
    - ex) 상수, 변수, 포인터
- **가변 공간**: 입력 크기에 따라 메모리 사용량이 달라짐
    - 배열, 리스트, 재귀 스택

<br>

### 2. 공간 복잡도 종류

| 표기 | 이름 | 의미 | 예시 |
| --- | --- | --- | --- |
| **O(1)** | 상수 공간 | 입력 크기와 관계없이 일정한 메모리 사용 | 단일 변수 사용 |
| **O(log n)** | 로그 공간 | 재귀 호출 깊이에 비례한 스택 사용 | 이진 탐색(재귀) |
| **O(n)** | 선형 공간 | 입력 크기만큼 추가 저장 공간 필요 | 배열 복사, 리스트 생성 |
| **O(n²)** | 제곱 공간 | 2차원 구조 사용 | n×n 행렬 생성 |

<br>

### 3. 중요 이유

- 메모리가 제한된 환경에서는 불필요한 공간 사용이 성능 저하로 이어진다.
- 함수 호출 시 스택이 누적되므로 공간 복잡도 분석이 필수적이다.

<br>

### 4. 시간 복잡도 vs 공간 복잡도

| 구분 | 시간 복잡도 | 공간 복잡도 |
| --- | --- | --- |
| **정의** | 알고리즘의 실행 시간 분석 | 알고리즘의 메모리 사용량 분석 |
| **목적** | 처리 속도 최적화 | 메모리 효율성 최적화 |
| **영향 요인** | 반복문, 재귀 호출, 연산 수 | 변수, 자료 구조, 스택 사용량 |
| **표기법** | O(1), O(n), O(n log n), O(n²)… | O(1), O(n), O(n²)… |
| **관계** | 공간을 더 쓰면 시간을 줄일 수 있음 | 시간을 더 쓰면 공간을 절약할 수 있음 |

<br>

## 점근적 표기법

**점근적 표기법**은 알고리즘의 실행 시간이나 메모리 사용랑이  **입력 크기 n이 커질 때** 어떤 속도로 증가하는지를 표현하는 수학적 표기법이다.

**상수나 작은 항들은 무시하고, 증가율에만 집중한다.**

<br>

| 표기 | 이름 | 의미 | 설명 |
| --- | --- | --- | --- |
| **O( )** | 빅오(Big-O) | **상한 (최악의 경우)** | 실행 시간이 이 값보다 느리지 않음을 보장 |
| **Ω( )** | 오메가(Omega) | **하한 (최선의 경우)** | 실행 시간이 이 값보다 빠르지 않음을 보장 |
| **Θ( )** | 세타(Theta) | **평균** | 상한과 하한이 일치할 때 사용 |
