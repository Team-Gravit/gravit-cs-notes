## 기수 정렬(Radix Sort)

**낮은 자리수부터 비교**하여 정렬하는 알고리즘이다. 다른 정렬 알고리즘과 달리 **비교 연산을 수행하지 않고** 자릿수를 이용해 정렬한다. **안정 정렬**이며, 자릿수가 고정된 정수나 문자열 정렬에 효율적이다.

<br>

### 1. 특징

- **비교 연산을 수행하지 않는 정렬**(Non-Comparison Sort)
- **낮은 자리수부터 높은 자리수**로 순차적으로 정렬함(LSD - Least Significant Digit)
- **안정 정렬**(Stable Sort), 동일한 값의 순서가 유지됨
- **0~9까지의 버킷**(Bucket) 또는 큐(Queue) 사용
- 정렬할 수들의 **최대 자릿수**(d)에 영향을 받음
- 내부적으로 **계수 정렬**(Counting Sort)을 활용함

<br>

### 2. 알고리즘 동작 과정

**버킷 준비**

0부터 9까지 **10개의 버킷**(큐)을 준비한다.

<br>

**분배(Distribute)**

**현재 자릿수**를 기준으로 각 데이터를 해당 버킷에 넣는다.

<br>

**수집(Collect)**

**0번 버킷부터 순서대로** 데이터를 다시 가져온다.

<br>

**반복**

**가장 높은 자릿수**까지 분배~수집 과정을 반복한다.

<br>

### 4. 버킷 시스템

기수 정렬에서 **버킷**은 각 자릿수의 값(0~9)에 해당하는 **임시 저장 공간**이다.

<br>

**버킷의 역할**

```
버킷 인덱스 = (숫자 / 10^현재자리) % 10

예시:
- 숫자 802, 1의 자리 → (802 / 1) % 10 = 2 → 버킷 2
- 숫자 802, 10의 자리 → (802 / 10) % 10 = 0 → 버킷 0
- 숫자 802, 100의 자리 → (802 / 100) % 10 = 8 → 버킷 8

```

<br>

**버킷 구현 방식**

| **방식**      | **설명**                  | **특징**                 |
| ------------- | ------------------------- | ------------------------ |
| **큐**        | FIFO 구조로 순서 유지     | 구현 직관적              |
| **계수 정렬** | 카운팅 배열 + 누적합 사용 | 메모리 효율적, 속도 빠름 |

<br>

### 5. 시간 복잡도

| **경우**           | **시간 복잡도** |
| ------------------ | --------------- |
| **최선/평균/최악** | O(d × (n + b))  |

**시간 복잡도 분석**

- **d**: 최대 자릿수(digit)
- **n**: 정렬할 데이터의 개수
- **b**: 기수(radix, 10진법에서는 10)

<br>

**각 자릿수마다 수행하는 작업**

모든 데이터를 버킷에 분배하는 데 **O(n)**, 버킷에서 데이터를 다시 수집하는 데 **O(n + b)**(b개의 버킷 순회)가 소요된다. 이 과정을 d번 반복하므로 **O(d × (n + b))** 이다.

<br>

**실제 시간 복잡도**

일반적으로 b(기수)는 상수로 취급하므로 **O(d × n)** 또는 **O(dn)** 으로 표현한다.

<br>

### 6. 공간 복잡도

- **버킷 공간**: O(n + b) - n개의 데이터를 담을 출력 배열 + b개의 카운팅 배열
- **일반적인 경우**: O(n) - 기수 b(=10)는 상수로 취급

<br>

### 7. 기수 정렬의 장점

- **빠른 속도**: 자릿수가 적을 때 O(n)에 가까운 성능
- **안정 정렬**: 동일한 값의 순서가 유지됨
- **예측 가능한 성능**: 입력 데이터의 분포에 영향을 받지 않음
- **단순한 구조**: 비교 연산이 없어 구현이 직관적함

<br>

### 8. 기수 정렬의 단점

- **제한적 사용**: 정수나 고정 길이 문자열에만 사용 가능함
- **높은 공간 복잡도**: O(n + b)의 추가 메모리 필요
- **자릿수 의존성**: 자릿수가 클수록 성능 저하됨
- **부동소수점/음수 처리**: 추가적인 전처리 필요함

<br>

### 9. 적용 제약 사항

**사용 가능한 경우**

- 정수 데이터
- 고정 길이 문자열
- 자릿수가 적은 데이터

<br>

**사용 불가능한 경우**

- 부동소수점 숫자(전처리 필요)
- 음수(오프셋 처리 필요)
- 가변 길이 데이터
- 비교 기반 정렬이 필요한 경우

<br>

### 10. 다른 정렬 알고리즘과 비교

| **특징**        | **기수 정렬** | **퀵 정렬**           | **병합 정렬**         | **힙 정렬**           |
| --------------- | ------------- | --------------------- | --------------------- | --------------------- |
| **시간 복잡도** | O(d × n)      | O(n log n)            | O(n log n)            | O(n log n)            |
| **공간 복잡도** | O(n + b)      | O(log n)              | O(n)                  | O(1)                  |
| **안정성**      | 안정          | 불안정                | 안정                  | 불안정                |
| **비교 연산**   | 없음          | 있음                  | 있음                  | 있음                  |
| **적용 범위**   | 정수, 문자열  | 모든 비교 가능 데이터 | 모든 비교 가능 데이터 | 모든 비교 가능 데이터 |
| **자릿수 영향** | 크게 받음     | 받지 않음             | 받지 않음             | 받지 않음             |
