## 브루트 포스

가능한 모든 경우의 수를 탐색하면서 조건을 만족하는 해만 고르는 알고리즘이다.

<br>

### 1. 브루트 포스 vs 완전 탐색
두 방식은 같은 의미로 쓰이기도 하지만, 미세한 차이가 있다.

**브루트 포스**는 **결과를 찾는 것**에 중점을 두고 있고, **완전 탐색**은 **탐색**에 중점을 두고 있다.

<br>

### 2. 구현

**반복문 기반**

경우의 수가 일정하거나 단순할 때 사용한다. 
    
```python
for (int i = 0; i < 1000; i++) {
for i in range(1000):
    if check(i):
        print(f"정답: {i}")
        break
```

<br>
    
**재귀 기반**

경우의 수가 단계적으로 늘어나거나 깊이 있는 탐색에서 사용한다.

```python
nums = [0, 1]
path = []

def dfs(depth):
    if depth == 2:
        print(path)
        return
    for num in nums:
        path.append(num)
        dfs(depth + 1)
        path.pop()

dfs(0)
```

<br>

### 3. 브루트 포스의 장점
- 알고리즘의 설계와 구현이 간단
- 모든 경우의 수를 탐색하기 때문에 해를 보장함

<br>

### 4. 브루트 포스의 단점
- 모든 경우의 수를 탐색하기 때문에 시간이 매우 오래 걸림
- 메모리 사용이 비효율적임

<br>

### 5. 한계
브루트 포스는 모든 경우의 수를 탐색하기 때문에 시간적, 공간적 비용이 크다. 따라서 모든 경우를 탐색하지 않고, **필요한 부분만 선택적으로 확인**하는 **백트래킹**을 대안으로 사용할 수 있다.
