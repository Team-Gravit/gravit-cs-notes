# 운영체제 개요(Operating System Overview)

운영체제는 하드웨어와 사용자 사이의 **인터페이스 역할**을 수행하며, 컴퓨터 시스템의 **자원을 효율적으로 관리**하는 소프트웨어이다. 운영체제는 **커널**(Kernel)을 핵심으로 하여 프로세스, 메모리, 파일, 입출력 장치 등을 제어하고 사용자에게 편리한 환경을 제공한다.

<br>

### 1. 운영체제의 정의

운영체제(Operating System, OS)는 컴퓨터 하드웨어와 응용 프로그램 사이에서 **중재자** 역할을 하는 시스템 소프트웨어이다.

<br>

### **운영체제의 구성 요소**

**하드웨어**

- CPU, 메모리, 디스크, 키보드, 마우스, 네트워크 카드 등 물리적 자원

**운영체제**

- 커널, 시스템 프로그램, 장치 드라이버 등

**응용 프로그램**

- 워드프로세서, 웹 브라우저, 게임 등 사용자가 실행하는 프로그램

**사용자**

- 응용 프로그램을 통해 컴퓨터를 사용하는 주체

<br>

### 2. 운영체제의 목적

| **목적** | **설명** |
| --- | --- |
| **처리능력 향상** | CPU가 주어진 시간 내에 더 많은 작업을 처리하도록 함 |
| **사용 가능도 향상** | 시스템을 더 오랜 시간 동안 안정적으로 사용 가능하게 함 |
| **신뢰도 향상** | 시스템이 주어진 문제를 정확하게 해결하도록 보장 |
| **반환 시간 단축** | 사용자의 요청에 대한 응답 시간을 최소화 |

<br>

### 3. 운영체제의 역할

### 3-1. 자원 관리

운영체제는 한정된 시스템 자원을 효율적으로 배분하고 관리한다.

**관리 대상 자원**

- **CPU**: 프로세스 스케줄링을 통해 CPU 시간 할당
- **메모리**: 주기억장치 공간 할당 및 회수
- **입출력 장치**: 디스크, 프린터, 키보드 등의 장치 관리
- **파일**: 파일 시스템을 통한 데이터 저장 및 접근 관리

<br>

### 3-2. 사용자 지원

사용자와 하드웨어 사이의 **매개체** 역할을 수행한다.

- 사용자 명령을 해석하고 실행
- 사용자 인터페이스(CLI, GUI) 제공
- 응용 프로그램 실행 환경 제공

<br>

### 3-3. 하드웨어 보호

응용 프로그램이 하드웨어를 직접 제어하는 것을 방지하여 시스템의 **안정성**을 확보한다.

<br>

### 4. 커널(Kernel)

**커널**은 운영체제의 핵심 요소로, 항상 메모리에 상주하며 하드웨어와 응용 프로그램 사이의 **가교 역할**을 수행한다.

<br>

### **커널의 주요 기능**

| **기능** | **설명** |
| --- | --- |
| **보안** | 하드웨어와 프로세스의 보안 관리 |
| **자원 관리** | 한정된 시스템 자원의 효율적 관리 |
| **추상화** | 하드웨어 복잡성을 숨기고 일관된 인터페이스 제공 |

<br>

### **커널의 종류**

**1) 일체형 커널(Monolithic Kernel)**

운영체제의 모든 서비스가 커널 내부에 포함된 구조

<img width="2162" height="1491" alt="Frame 1171275951 (2)" src="https://github.com/user-attachments/assets/b1482d07-3788-4ed3-bc73-63fda66d185a" />


- **장점**: 커널 내부 구성 요소 간 효율적인 상호작용
- **단점**: 하나의 요소에서 오류 발생 시 시스템 전체에 영향
- **예시**: UNIX, Linux

<br>

**2) 마이크로 커널(Microkernel)**

최소한의 기능만 커널에 포함하고 나머지는 사용자 공간으로 분리한 구조

<img width="2162" height="1478" alt="Frame 1171275949" src="https://github.com/user-attachments/assets/73722cf3-739d-4694-aad1-ba8af1b192a7" />


- **장점**: 확장성과 안정성이 높음, 유지보수 용이
- **단점**: IPC로 인한 성능 저하
- **예시**: Minix, QNX

<br>

### 5. 동작 모드

운영체제는 시스템의 안정성을 위해 두 가지 동작 모드를 제공한다.

<br>

### 5-1. 커널 모드(Kernel Mode) = 슈퍼바이저 모드

- **특징**: CPU가 모든 명령어를 실행할 수 있는 모드
- **동작**: 운영체제 커널이 실행되는 모드
- **권한**: 하드웨어를 직접 제어 가능

<br>

### 5-2. 사용자 모드(User Mode) = 보호 모드

- **특징**: 제한된 명령어만 실행 가능한 모드
- **동작**: 응용 프로그램이 실행되는 모드
- **권한**: 하드웨어 직접 제어 불가, 시스템 콜을 통해 운영체제에 요청

<br>

### 6. 운영체제의 구성

운영체제는 자원의 성격에 따라 네 가지 주요 관리자로 구성된다.

<br>

### 6-1. 프로세스 관리자(Process Manager)

- 프로세스 생성, 삭제, 상태 관리
- CPU 스케줄링을 통한 프로세스 실행 순서 결정
- 프로세스 상태 전이 처리 (준비, 실행, 대기 등)

<br>

### 6-2. 메모리 관리자(Memory Manager)

- 주기억장치 공간 할당 및 회수
- 메모리 사용 현황 추적 및 테이블 관리
- 운영체제 자신의 메모리 영역 보호

<br>

### 6-3. 장치 관리자(Device Manager)

- 모든 하드웨어 장치의 할당, 작동, 반환 관리
- 장치 스케줄링을 통한 효율적 자원 분배
- 장치 드라이버를 통한 하드웨어 제어

<br>

### 6-4. 파일 관리자(File Manager)

- 시스템의 모든 파일 관리
- 파일 접근 권한 제어
- 파일 열기/닫기를 통한 자원 할당/회수

<br>

### 7. 운영체제의 유형

### 7-1. 일괄 처리 시스템(Batch Processing System)

- 작업을 모아서 한꺼번에 처리
- 천공 카드나 자기 테이프를 입력 장치로 사용
- **평가 지표**: 처리량(Throughput), 반환시간(Turnaround Time)

<br>

### 7-2. 시분할 시스템(Time Sharing System)

- 여러 사용자가 동시에 시스템을 대화형으로 사용
- CPU 시간을 작은 단위로 분할하여 여러 프로세스에 할당
- **평가 지표**: 응답 시간(Response Time)

<br>

### 7-3. 다중 처리 시스템(Multiprocessing System)

- 여러 개의 프로세서를 사용하여 병렬 처리
- **SMP(Symmetric Multiprocessing)**: 모든 프로세서가 동등한 역할
- **처리량 증가**: 프로세서 수에 비례하여 성능 향상

<br>

### 7-4. 실시간 시스템(Real-Time System)

- 시간 제약이 있는 작업을 처리
- **경성 실시간 시스템**: 시간 제약을 반드시 지켜야 함 (예: 항공기 제어)
- **연성 실시간 시스템**: 시간 제약이 다소 유연함 (예: 멀티미디어 스트리밍)

<br>

### 7-5. 분산 시스템(Distributed System)

- 네트워크로 연결된 여러 컴퓨터가 하나의 시스템처럼 동작
- 자원 공유 및 부하 분산
- **예시**: 클라우드 컴퓨팅, 클러스터 시스템

<br>

### 8. 주요 운영체제의 특징

| **운영체제** | **커널 유형** | **특징** |
| --- | --- | --- |
| **UNIX** | 일체형 커널 | 다중 사용자, 시분할, 이식성 높음 |
| **Linux** | 일체형 커널 | 오픈 소스, 무료, 다양한 배포판 |
| **Windows** | 하이브리드 커널 | GUI 중심, 광범위한 응용 프로그램 지원 |
| **macOS** | 하이브리드 커널 | BSD 기반, Darwin 커널, 사용자 친화적 |
| **Android** | 일체형 커널(Linux) | 모바일 특화, 터치 인터페이스 |

<br>

### 9. 장단점

### **운영체제의 장점**

- 하드웨어 자원의 효율적 관리
- 사용자에게 편리한 인터페이스 제공
- 시스템의 안정성과 보안 보장
- 다양한 응용 프로그램 실행 환경 제공

<br>

### **운영체제의 단점**

- 시스템 오버헤드 발생 (메모리 사용, CPU 시간 소모)
- 커널 오류 시 시스템 전체에 영향
- 복잡한 구조로 인한 개발 및 유지보수 어려움

<br>

### 11. 관련 개념

- **시스템 콜(System Call)**: 응용 프로그램이 운영체제 서비스를 요청하는 인터페이스
- **인터럽트(Interrupt)**: 하드웨어나 소프트웨어가 CPU에게 즉각적인 처리를 요청하는 메커니즘
- **부트스트랩(Bootstrap)**: 컴퓨터 전원을 켤 때 운영체제를 메모리에 적재하는 과정
- **가상화(Virtualization)**: 물리적 자원을 논리적으로 분할하여 여러 운영체제가 동시에 실행되도록 하는 기술
