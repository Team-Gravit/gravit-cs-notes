## 연결리스트(Linked List)

각 노드가 **데이터**와 **다음 노드의 주소**를 함께 저장하는 자료구조. 배열과 달리 **논리적 저장 순서**와 **물리적 저장 순서**가 일치하지 않아 삽입/삭제가 자유롭지만, 인덱스가 없어 탐색 시 **순차 접근**이 필요하다.

<br>

### 1. 특징

- **노드**는 **데이터 영역**과 **포인터**(다음 노드 주소)로 구성
- 크기가 고정되지 않아 **동적 확장** 가능
- 메모리 상에서 노드들이 **비연속적**으로 배치

<br>

### 2-1. 연결리스트의 장점

- **삽입**/**삭제** 시 포인터 영역만 변경하므로 **O(1)** 시간 복잡도 소요
- 크기 제약 없이 **동적 확장** 가능

<br>

### 2-2. 연결리스트의 단점

- 인덱스가 없어 특정 요소 **탐색**에 **O(n)** 시간 복잡도 소요
- 포인터 저장을 위한 **추가 메모리** 필요

<br>

### 3. 연결리스트의 삽입 과정

노드 삽입 시 포인터만 조정하면 되므로 배열보다 효율적이다.

```
[A] → [B] → [D]

      ↓ C 삽입

[A] → [B] → [C] → [D]

1. C의 next를 D로 설정
2. B의 next를 C로 변경
```

**시간 복잡도**: 삽입 위치를 아는 경우 **O(1)**, 탐색 포함 시 **O(n)**

<br>

### 4. 연결리스트의 삭제 과정

삭제할 노드의 이전 노드가 다음 노드를 가리키도록 포인터를 변경한다.

```
[A] → [B] → [C] → [D]

      ↓ B 삭제

[A] → [C] → [D]

1. A의 next를 C로 변경
```

**시간 복잡도**: 삭제 위치를 아는 경우 **O(1)**, 탐색 포함 시 **O(n)**

<br>

### 5. 배열과 비교

| **연산** | **배열** | **연결리스트** |
| --- | --- | --- |
| **접근** | O(1) | O(n) |
| **탐색** | O(n) | O(n) |
| **삽입** | O(n) | O(1)* |
| **삭제** | O(n) | O(1)* |

*위치를 아는 경우. 탐색 포함 시 O(n)

<br>

**선택 기준**

- **탐색**/**조회**가 빈번 → **배열**
- **삽입**/**삭제**가 빈번 → **연결리스트**

<br>

### 6. 연결리스트의 종류

**단일 연결리스트(Singly Linked List)**

- 각 노드가 다음 노드만 가리킴
- 단방향 순회만 가능

<br>

**이중 연결리스트(Doubly Linked List)**

- 각 노드가 이전/다음 노드 모두 가리킴
- 양방향 순회 가능

<br>

**원형 연결리스트(Circular Linked List)**

- 마지막 노드가 첫 노드를 가리킴
- 순환 구조
