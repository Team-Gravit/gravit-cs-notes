## 힙(Heap)

**완전 이진 트리** 기반의 자료구조로, **최대값** 또는 **최소값**을 빠르게 찾기 위해 설계되었다. **우선순위 큐**를 구현하는 가장 효율적인 방법이다. **형제 노드** 간의 우선순위는 고려하지 않으며, **부모 노드**와 **자식 노드** 간의 대소관계만 유지하는 **반정렬 상태**이다.

<br>

### 1. 힙의 종류

<img src="https://raw.githubusercontent.com/Team-Gravit/gravit-images/main/data-structure/unit06/image1.png" width="50%">

**최대 힙(Max Heap)**

부모 노드의 값이 자식 노드의 값보다 **크거나 같은** 완전 이진 트리다. 루트 노드에 **최대값**이 위치하므로 **O(1)** 시간에 최대값 접근이 가능하다.

<br>

**최소 힙(Min Heap)**

부모 노드의 값이 자식 노드의 값보다 **작거나 같은** 완전 이진 트리다. 루트 노드에 **최소값**이 위치하므로 **O(1)** 시간에 최소값 접근이 가능하다.

<br>

### 2. 힙의 특징

- **완전 이진 트리 구조**로 **배열**을 사용하여 효율적으로 구현할 수 있음
- 주로 배열의 **1번 인덱스**부터 사용함
- **중복된 값을 허용**하며, 형제 노드 간의 대소 관계는 정의하지 않음

<br>

### 3. 힙의 배열 표현

힙을 배열로 구현하는 경우, 부모와 자식 노드 간의 인덱스 관계는 다음과 같다.

- **왼쪽 자식 인덱스 = 부모 인덱스 X 2**
- **오른쪽 자식 인덱스 = ( 부모 인덱스 X 2 ) + 1**
- **부모 인덱스 = 자식 인덱스 % 2**

<br>

### 4. 힙의 삽입 과정

새로운 요소를 힙의 **마지막 위치**에 삽입한 뒤, **부모 노드와 비교**하며 힙 속성을 만족할 때까지 **위로 이동**시킨다.

<img src="https://raw.githubusercontent.com/Team-Gravit/gravit-images/main/data-structure/unit06/image2.png" width="75%">

<br>

### 5. 힙의 삭제 과정

**루트 노드**를 삭제한 뒤 **마지막 노드를 루트 위치로 이동**시킨다. 그 다음 자식 노드와 비교하며 힙 속성을 만족할 때까지 **아래로 이동**시킨다.

<br>

아래로 이동할 때 **최대 힙**인 경우 자식 노드 중 **더 큰 노드와 비교**하면서 교환하고, **최소 힙**인 경우에는 자식 노드 중 **더 작은 노드**와 비교하면서 교환한다.

<br>

<img src="https://raw.githubusercontent.com/Team-Gravit/gravit-images/main/data-structure/unit06/image3.png" width="75%">

<br>

### 6. 우선순위 큐

**우선순위 큐**는 **우선순위가 높은 데이터가 먼저 나가는 자료구조**다. 배열이나 연결리스트로도 구현이 가능하지만, 힙을 사용하면 **삽입**/**삭제** 모두 O(log n)으로 효율적으로 구현할 수 있다.
